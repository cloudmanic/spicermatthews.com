## Development Commands

### CSS/Styling

- Build TailwindCSS: `npx tailwindcss -i resources/css/style.css -o static/css/build.css`
- TailwindCSS watches `./**/*.{html,js}` files for changes

### Hugo Development

- Start development server: `hugo server -D` (using `./hugo` binary in repo)
- Build for production: `hugo` (using `./hugo` binary in repo)
- When running hugo we use the local binary instead of the global one `./hugo`

## Architecture

This is a Hugo static site generator blog for Spicer Matthews' personal website:

- **Hugo Configuration**: Uses `config.toml` with `publishDir = "docs"` for GitHub Pages deployment
- **Content Structure**: Blog posts in `content/blog/` as HTML files, not markdown
- **Styling**: TailwindCSS with custom theme colors (`siteBackground: "#4A4A4A"`, `svgFillIcon`)
- **Layouts**:
  - Base template in `layouts/_default/baseof.html`
  - Homepage in `layouts/index.html`
  - Blog-specific layouts in `layouts/blog/`
- **Static Assets**: Images and demos in `static/` directory
- **Build Output**: Generated site goes to `docs/` directory for GitHub Pages

### Key Files

- `config.toml`: Hugo configuration with GitHub Pages setup
- `tailwind.config.js`: TailwindCSS configuration with custom theme
- `layouts/_default/baseof.html`: Main HTML structure with partials
- `content/blog/`: Blog posts (HTML format, not markdown)
- `static/`: Static assets (images, CSS, demos)

## Image Generation (nano-banana)

Use `/opt/homebrew/bin/nano-banana` to generate images. It's already authenticated with the Gemini provider (credentials at `~/.config/gemini/credentials`). Default provider is `gemini`.

Always use `gemini-3-pro-image-preview` as the model — it produces the best quality.

### Two invocation modes

**With a style preset (recommended for consistent output):**

```bash
/opt/homebrew/bin/nano-banana \
  -model gemini-3-pro-image-preview \
  -c /path/to/preset.md \
  -o /absolute/path/output.jpg \
  "subject-only description"
```

**Without a preset (one-off image):**

```bash
/opt/homebrew/bin/nano-banana \
  -model gemini-3-pro-image-preview \
  -o /absolute/path/output.png \
  -s 1920x1080 \
  "full description including style, framing, and any constraints"
```

### Key flags

- `-model` — model override. Always pass `gemini-3-pro-image-preview`.
- `-c` / `-config` — path to a preset `.md` file (frontmatter + style sections).
- `-o` / `-out` — output file path. Use absolute paths. Extension (`.jpg` / `.png`) implies format.
- `-s` / `-size` — `WxH` (e.g. `1920x1080`). Only needed when not using a preset. Default `1024x1024`.
- `-f` / `-format` — `png` or `jpeg`. Usually inferred from `-o`.
- `-q` / `-quality` — JPEG quality 1–100. Default 92.
- `-fit` — `cover` (default) or `contain`.
- Final positional arg — the prompt. With a preset, keep this focused on the subject; the preset handles style.

### Preset file format

```markdown
---
format: jpeg
quality: 92
size: 1024x1024
---

## Style
(visual style, palette, medium)

## Framing
(composition, angle, negative space)

## Avoid
(things to exclude — text, photorealism, clutter, etc.)

## Context
(what the image is for, so the model can calibrate tone)
```

When Spicer asks for an image "in the [Preset Name] style," pass the preset with `-c` and use the prompt for the subject only.

### Output location

Write generated images to `~/.claude/image-cache/<some-dir>/` (a gitignored cache), NOT to the project root. Create a subdirectory per task if generating multiple related images. Copy finalized blog images into `static/` once approved.

### Verifying output

After generating, use the Read tool on the output file to view the image and confirm it matches the request. If it doesn't, iterate on the prompt or preset.

### Other

- `nano-banana auth` — interactively store an API key for a provider (already done for Gemini).
- Default timeout is 2 minutes; pass `-timeout 5m` for complex generations.
